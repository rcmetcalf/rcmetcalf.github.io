     This module about information structures taught me that computers use specific languages to build webpages, write documents, and follow instructions. Everything we see on a webpage is text processed and structured by the computer.

Even this web page is all text! See:
     Extenisble markup language, or XML, is the framework used to create all other types of coding languages, like hyper text markup language (HTML) and cascading style sheets (CSS). In the photo above, I used HTML to create a structure for this webpage, but I also used XML. The doctype declaration at the top of the image is XML, as well as the title and links. The declaration tells the computer that this file should be read and rendered as an HTML file. The title defines the webpage's name, and the links are associated with the CSS document and imported fonts. This text is not seen on the page, but the computer uses this metadata to build the page correctly.

Notice the comment at the top.

     XML, HTML, and CSS all have specific standards of use on the web. As Dr. Berquist said, it is helpful to know the web standards "so that we can know when and why we might have to deviate from them." Web standards apply to both the structural layer (HTML and XML) and the presentational layer (CSS).
